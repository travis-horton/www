The D Flip-Flop, pt 1

So, computers all begin with little chips. Actually they start with little transistors. Nowadays they are very, very little. Like 70 ATOMS wide.

Those little teeny tiny transistors make up what are called gates. A gate is the most basic logic unit in a computer. Here's an example of how transistors make up an AND gate:

The "V" at the top of the image is the source of the current and the little striped triangle (aka ground) at the bottom of the image is the where the electricity wants to go. It always wants to get from the "V" to ground.

The two circles with little symbols in them are transistors. You can watch a video on how they work if you're curious.

"A" and "B" are the two inputs and "Out" is the output. What we want to get out is current if and only if both "A" and "B" have current. (Another way to say this is that "A" and "B" are on, or true, or "1"s.) Transitors only allow current through if the input has current, so in this case, we only get current at the output if both "A" and "B" have current.

There a few other types of gates: OR gates, NOT gates, NAND gates, and several others. Go read about them here.

With these gates you can build all sorts of more complicated logical units. You can bunch them together and have "buses", which are just several of these simple gates in parallel with the expected number of outputs. You can build arithmetic logical units to compute basic arithmetic operations.

This brings us up to Chapter 3 of Nand2Tetris. Up to this chapter, everything the authors have used as givens felt like legitimate givens. However, in this chapter they introduce the Flip-Flop. The Flip-Flop is basic building block of all memory operations. I'll quote their description (their italics):

Flip-Flops The most elementary sequential element in the computer is a device called a flip-flop, of which there are several variants. In this book we use a variant called a data flip-flop, or DFF, whose interface consists of a single-bit data input and a single-bit data output. In addition, the DFF has a clock input that continuously changes according to the master clock's signal. Taken together, the data and the clock inputs enable the DFF to implement the time-based behavior out(t) = in(t - 1), where in and out are the gate's input and output values and t is the current clock cycle. In other words, the DFF simply outputs the input value from the previous time unit.

And yet, later:

The physical implementation of a DFF is an intricate task, and is based on connecting several elementary logic gates using feedback loops (one classic design is based on NAND gates alone). In this book we have chosen to abstract away this complexity, treating DFFs as primitive building blocks.

Well, that's not good enough for me. Why should I build a computer from little transistors and yet not truly understand the basic building block of memory.

So, that's what this blog post is all about, but I think this post actually turned into simply building a pre-DFF foundation.

A little code:

CHIP Not {
IN in;
OUT out;

PARTS:
Nand(a=in, b=in, out=out);
}
